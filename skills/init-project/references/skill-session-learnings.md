# Session Learnings

Reflect on the current coding session, identify reusable lessons, and persist them so future sessions avoid the same pitfalls.

## Step 1: Identify learnings

Review the conversation for:

- Errors that required multiple attempts to fix
- Tooling surprises (e.g. auto-generated files, deprecated APIs)
- Project-specific conventions that weren't obvious
- Patterns that worked well and should be repeated

For each candidate, ask:

- **Is it already enforced?** If a linter, type checker, or existing rule catches it, skip — the tool teaches at runtime.
- **Is it project-specific?** Generic best practices don't belong. Only capture knowledge an agent couldn't infer from the code alone.
- **Is it durable?** Avoid time-sensitive notes. If it will change soon, skip it.

## Step 2: Persist the learning

**Default: append to project `CLAUDE.md`**

Add a new `##` section to the project's root `CLAUDE.md`. This is the single source of truth for project conventions.

- Keep it to 1-4 sentences. Be direct.
- Include a short code example only if the correct pattern is non-obvious.
- Don't repeat what linters or type checkers already enforce.
- Don't duplicate existing entries — check first.

**Alternative: create a new skill**

If the learning is a multi-step workflow with decision points or templates, create a skill in `.claude/skills/<skill-name>/SKILL.md` instead.

## Step 3: Verify

- Read the updated `CLAUDE.md` back to confirm it merged cleanly.
- Confirm no duplicate or contradictory entries exist.

## Example

A session revealed that `routeTree.gen.ts` is auto-generated and shouldn't be manually regenerated. This is project-wide and not caught by any linter, so append to `CLAUDE.md`:

```markdown
## TanStack Router route generation

The route tree (`src/routeTree.gen.ts`) is auto-generated by the TanStack Router Vite plugin. When you create a new route file under `src/routes/`, do **not** try to manually regenerate it with CLI commands. The plugin regenerates it automatically when the dev server is running or on the next build.
```
