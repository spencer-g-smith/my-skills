# Weekly Maintenance Runbook

Execute once per week. Two required outputs:
1) `Weekly/YYYY-MM-DD - Knowledge Base Maintenance.md`
2) `Weekly/YYYY-MM-DD - Weekly Recap + Next Week Plan.md`

## Design Goals
- Incremental updates (not full reprocessing)
- Clean searchable life record
- Decision-oriented weekly synthesis of Spencer's week

## Canonical Paths
For source/insight/application destinations, follow:
- `references/knowledge-base-governance.md`

Additional paths used by this runbook:
- Daily reports: `Daily/Reports/`
- Weekly outputs: `Weekly/`

## Section A — Knowledge Base Maintenance
1. Select new/changed reading sources since last run.
2. Distill/update insights (dedupe first).
3. Route useful applications into Self-Development and Coupa Roundup.
4. Update links and minimal index/MOC entries when changed.
5. Write `Weekly/YYYY-MM-DD - Knowledge Base Maintenance.md` with:
   - processed sources
   - insights created/updated
   - carry-forward queue

## Section B — Weekly Recap + Next Week Plan (Spencer-focused)
### Inputs
- Previous 7 days of `Daily/Reports/`
- Same-week memory notes if present
- Current week KB maintenance report

### Rules
- Summarize **Spencer's week**, not agent activity.
- Prefer outcomes, decisions, meetings, personal/family signals, and open loops.
- Keep concise and searchable.

### Output format (simple)
Write: `Weekly/YYYY-MM-DD - Weekly Recap + Next Week Plan.md`
- Week summary
- What mattered most
- Open loops
- Next week focus

## Communication
- Send concise summary only when meaningful changes were made.
- If nothing material changed, stay silent.

## Definition of Done
- Both weekly files exist.
- Weekly recap is Spencer-week focused.
- Link integrity preserved.
- Incremental state updated.
